# Doc Routing

Use these docs by task type:

- product or scope changes: `docs/prd.md`
- system and module boundaries: `docs/architecture.md`
- delivery planning: `docs/roadmap.md`
- unresolved decisions: `docs/open-questions.md`
- internal schema edits: `llmdoc/reference/canonical-schema.md`
- repo restructuring: `llmdoc/reference/repo-layout.md`
